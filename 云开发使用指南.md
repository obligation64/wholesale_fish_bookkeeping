# 云开发功能使用指南

## 快速开始

### 1. 安装依赖

```bash
cd /home/zhoumeihua/wholesale_fish_bookkeeping
npm install
```

### 2. 配置密钥(可选)

如果需要在本地服务器运行并使用云同步功能,需要配置腾讯云API密钥。

**获取密钥:**
1. 访问 https://console.cloud.tencent.com/cam/capi
2. 登录腾讯云账号
3. 创建或获取 API 密钥

**配置密钥:**

方式1 - 使用环境变量(推荐):
```bash
export TCB_SECRET_ID="你的SecretId"
export TCB_SECRET_KEY="你的SecretKey"
```

方式2 - 使用.env文件:
```bash
cp .env.example .env
# 编辑.env文件,填入密钥
```

### 3. 启动服务

```bash
npm start
```

服务启动后,会自动:
- 初始化本地数据库
- 尝试连接云开发
- 启动自动同步(每30秒一次)

## 使用场景

### 场景1: 手动同步数据

#### 全量同步(首次使用或数据丢失)
```bash
curl -X POST http://localhost:3000/api/cloud/sync/all
```

#### 增量同步(日常使用)
```bash
curl -X POST http://localhost:3000/api/cloud/sync/incremental
```

### 场景2: 查看同步状态

```bash
curl http://localhost:3000/api/cloud/sync/status
```

返回示例:
```json
{
  "success": true,
  "data": {
    "lastSyncTime": "2026-03-12T10:00:00.000Z",
    "syncing": false,
    "cloudInitialized": true,
    "statuses": {
      "customers": {
        "table_name": "customers",
        "last_sync_time": "2026-03-12T10:00:00.000Z"
      }
    }
  }
}
```

### 场景3: 从云端恢复数据

如果本地数据丢失或损坏,可以从云端恢复:

```bash
# 恢复客户数据
curl -X POST http://localhost:3000/api/cloud/restore/customers

# 恢复产品数据
curl -X POST http://localhost:3000/api/cloud/restore/products

# 恢复销售数据
curl -X POST http://localhost:3000/api/cloud/restore/sales

# ... 其他表
```

## 集合说明

系统会在云开发中创建以下数据集合:

| 集合名 | 说明 | 用途 |
|--------|------|------|
| customers | 客户信息 | 存储所有客户数据 |
| products | 产品信息 | 存储所有产品数据 |
| sales | 销售记录 | 存储所有销售记录 |
| expenses | 支出记录 | 存储所有支出记录 |
| monthly_expenses | 月度固定支出 | 存储月度固定支出 |
| inventory | 库存数据 | 存储库存信息 |
| change_logs | 变更日志 | 记录所有数据变更 |
| sync_status | 同步状态 | 记录各表的同步时间 |

## 自动同步

系统默认开启自动同步,每30秒自动执行增量同步。

### 修改同步间隔

编辑 `config.js` 文件:

```javascript
sync: {
    enabled: true,
    interval: 30000,  // 改为60000即1分钟
    batchSize: 100
}
```

### 关闭自动同步

```javascript
sync: {
    enabled: false,
    interval: 30000,
    batchSize: 100
}
```

## 常见问题

### Q1: 启动时提示"云开发同步初始化失败"

**原因:** 可能是缺少API密钥或网络问题

**解决方法:**
1. 如果在云函数内运行,不需要密钥,可以忽略此警告
2. 如果在本地运行,需要配置API密钥
3. 检查网络连接

### Q2: 同步失败,提示权限错误

**原因:** API密钥没有足够权限

**解决方法:**
1. 检查API密钥是否有CloudBase访问权限
2. 在腾讯云控制台确认密钥权限配置

### Q3: 数据不同步

**原因:** 可能是网络问题或自动同步未开启

**解决方法:**
1. 检查网络连接
2. 查看自动同步是否开启
3. 手动执行同步: `curl -X POST http://localhost:3000/api/cloud/sync/incremental`

### Q4: 如何在云函数中使用?

**方法:** 在云函数中不需要配置secretId和secretKey,系统会自动获取环境信息。

```javascript
// config.js中只配置环境ID即可
tcb: {
    env: '6162-abc-8g2fdo0a0e77ed86-1255526316'
}
```

## 安全建议

1. **保护密钥**
   - 不要将密钥提交到代码仓库
   - 使用环境变量或.env文件
   - 定期更换密钥

2. **数据备份**
   - 定期执行全量同步
   - 在云开发控制台备份重要数据

3. **访问控制**
   - 限制API访问权限
   - 使用HTTPS加密传输
