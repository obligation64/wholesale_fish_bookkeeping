# 存储模式切换功能 - 更新说明

## 🎯 更新概述

在原有的云开发集成基础上,新增了**存储模式切换功能**,系统现在支持三种存储模式,可以在本地SQLite和云开发之间灵活切换。

## ✅ 新增功能

### 1. 三种存储模式

- **local (本地模式)**: 仅使用本地SQLite数据库
  - 无需网络
  - 无需云密钥
  - 访问速度最快

- **cloud (云模式)**: 仅使用腾讯云开发
  - 数据云端存储
  - 支持多设备访问
  - 集中管理

- **hybrid (混合模式)**: 本地为主,云端同步
  - 本地快速访问
  - 自动云端备份
  - 支持离线使用

### 2. 存储管理器

新增 `storage-manager.js` 模块:
- 统一管理存储模式
- 模式切换功能
- 云存储初始化
- 存储状态查询

### 3. 数据提供者

新增 `data-provider.js` 模块:
- 统一的数据访问接口
- 根据存储模式自动路由
- 云本地数据格式转换
- 混合模式自动同步

### 4. API接口

新增存储模式管理接口:
- `GET /api/storage/mode` - 获取当前存储模式
- `POST /api/storage/mode` - 切换存储模式

## 🔧 配置方式

### 环境变量
```bash
# 本地模式
export STORAGE_MODE=local

# 云模式
export STORAGE_MODE=cloud

# 混合模式
export STORAGE_MODE=hybrid
```

### .env文件
```env
STORAGE_MODE=local  # 或 cloud 或 hybrid
```

### API接口
```bash
# 查看当前模式
curl http://localhost:3000/api/storage/mode

# 切换模式
curl -X POST http://localhost:3000/api/storage/mode \
  -H "Content-Type: application/json" \
  -d '{"mode":"hybrid"}'
```

## 📊 存储模式对比

| 特性 | local | cloud | hybrid |
|------|-------|--------|--------|
| 数据存储位置 | 本地SQLite | 腾讯云开发 | 本地 + 云端 |
| 需要网络 | 否 | 是 | 是(仅同步时) |
| 需要云密钥 | 否 | 是 | 是 |
| 数据备份 | 需手动备份 | 云端自动备份 | 云端自动备份 |
| 访问速度 | 最快 | 依赖网络 | 最快 |
| 多设备同步 | 不支持 | 支持 | 支持 |
| 离线使用 | 支持 | 不支持 | 支持 |

- 集中管理
- 实时协作
