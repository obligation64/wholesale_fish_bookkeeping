## 📋 前置条件检查

### 1. 检查 Node.js 是否安装

```bash
node --version
```

**如果显示版本号（如 v18.x.x）** → 继续

**如果显示 "command not found"** → 需要先安装 Node.js

#### 安装 Node.js

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install -y nodejs npm
```

**CentOS/RHEL:**
```bash
sudo yum install -y nodejs npm
```

**使用 NVM（推荐）:**
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
nvm install 18
nvm use 18
```

## 🔥 三种启动方式

### 方式一：一键启动脚本（最简单）

```bash
cd /home/zhoumeihua/wholesale_fish_bookkeeping
bash start_backend.sh
```

或使用中文版本：
```bash
bash 启动后端服务.sh
```

### 方式二：手动启动

```bash
cd /home/zhoumeihua/wholesale_fish_bookkeeping

# 安装依赖（首次运行）
npm install

# 启动服务
npm start
```

### 方式三：使用 PM2（生产环境）

```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start server.js --name fish-bookkeeping

# 查看状态
pm2 status

# 查看日志
pm2 logs fish-bookkeeping

# 停止服务
pm2 stop fish-bookkeeping
```

## 🛠️ 常见问题

### Q1: 端口 3000 被占用

**解决方案1：** 修改端口

编辑 `server.js`，将第 6 行改为：
```javascript
const PORT = process.env.PORT || 3001;  // 改为其他端口
```

**解决方案2：** 关闭占用端口的程序
```bash
# 查找占用 3000 端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>
```

### Q2: npm install 失败

```bash
# 清除缓存重试
npm cache clean --force
npm install
```

### Q3: 权限问题

```bash
# 更改项目目录权限
cd /home/zhoumeihua/wholesale_fish_bookkeeping
chmod +x start_backend.sh
```

### Q4: 数据库文件损坏

每次启动会自动备份，查看备份文件：
```bash
ls -lh fish_bookkeeping_backup_*.db
```

### Q5: 后台运行服务

```bash
# 使用 nohup 后台运行
nohup npm start > backend.log 2>&1 &

# 查看日志
tail -f backend.log

# 查找进程
ps aux | grep "node server.js"
```
