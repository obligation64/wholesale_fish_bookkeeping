# 存储模式切换功能

## 概述

系统支持三种存储模式,可以根据需求灵活切换:

- **local (本地模式)**: 仅使用本地SQLite数据库
- **cloud (云模式)**: 仅使用腾讯云开发数据库
- **hybrid (混合模式)**: 本地SQLite为主,自动同步到云端

## 存储模式对比

| 特性 | local | cloud | hybrid |
|------|-------|--------|--------|
| 数据存储位置 | 本地SQLite | 腾讯云开发 | 本地 + 云端 |
| 需要网络 | 否 | 是 | 是(仅同步时) |
| 需要云密钥 | 否 | 是 | 是 |
| 数据备份 | 需手动备份 | 云端自动备份 | 云端自动备份 |
| 访问速度 | 最快 | 依赖网络 | 最快 |
| 多设备同步 | 不支持 | 支持 | 支持 |
| 离线使用 | 支持 | 不支持 | 支持 |
| 数据安全性 | 依赖本地备份 | 云端高可用 | 本地+云端双重保障 |

## 配置存储模式

### 方法1: 环境变量(推荐)

```bash
# 本地模式
export STORAGE_MODE=local

# 云模式
export STORAGE_MODE=cloud

# 混合模式
export STORAGE_MODE=hybrid
```

### 方法2: .env文件

复制并编辑 .env 文件:

```bash
cp .env.example .env
```

编辑 .env 文件:

```env
# 存储模式配置
STORAGE_MODE=local  # 或 cloud 或 hybrid

# 腾讯云开发配置(仅在cloud或hybrid模式下需要)
TCB_SECRET_ID=your-secret-id-here
TCB_SECRET_KEY=your-secret-key-here
```

### 方法3: 运行时切换

使用API接口切换存储模式:

```bash
# 切换到本地模式
curl -X POST http://localhost:3000/api/storage/mode \
  -H "Content-Type: application/json" \
  -d '{"mode":"local"}'

# 切换到云模式
curl -X POST http://localhost:3000/api/storage/mode \
  -H "Content-Type: application/json" \
  -d '{"mode":"cloud"}'

# 切换到混合模式
curl -X POST http://localhost:3000/api/storage/mode \
  -H "Content-Type: application/json" \
  -d '{"mode":"hybrid"}'
```

## 各模式详细说明

### 1. 本地模式 (local)

**适用场景:**
- 开发测试环境
- 网络不稳定环境
- 不需要多设备同步
- 数据量较小

**特点:**
- ✅ 无需网络连接
- ✅ 无需配置云密钥
- ✅ 数据访问速度最快
- ❌ 需要手动备份数据
- ❌ 无法多设备同步
- ❌ 数据安全性依赖本地

**使用方式:**
```bash
export STORAGE_MODE=local
npm start
```

### 2. 云模式 (cloud)

**适用场景:**
- 多人协作
- 需要远程访问
- 不关心本地存储
- 希望数据集中管理

**特点:**
- ✅ 数据云端存储
- ✅ 支持多设备访问
- ✅ 云端自动备份
- ✅ 云开发控制台管理
- ❌ 依赖网络连接
- ❌ 需要配置API密钥
- ❌ 访问速度依赖网络

**使用方式:**
```bash
export STORAGE_MODE=cloud
export TCB_SECRET_ID="your-secret-id"
export TCB_SECRET_KEY="your-secret-key"
npm start
```

### 3. 混合模式 (hybrid)

**适用场景:**
- 日常使用 + 云端备份
- 需要离线工作能力
- 数据安全要求高
- 需要多设备同步

**特点:**
- ✅ 本地访问,速度快
- ✅ 云端自动备份
- ✅ 支持离线使用
- ✅ 多设备数据同步
- ✅ 本地+云端双重保障
- ❌ 需要配置API密钥
- ⚠️  需要30秒定时同步

**使用方式:**
```bash
export STORAGE_MODE=hybrid
export TCB_SECRET_ID="your-secret-id"
export TCB_SECRET_KEY="your-secret-key"
npm start
```

## 使用示例

### 场景1: 日常使用(混合模式)

```bash
# 设置混合模式
export STORAGE_MODE=hybrid

# 启动服务
npm start
```

此时系统会:
1. 本地使用SQLite,访问速度快
2. 每30秒自动同步到云端
3. 即使断网也能正常使用
4. 恢复网络后自动续传

### 场景2: 开发测试(本地模式)

```bash
# 设置本地模式
export STORAGE_MODE=local

# 启动服务
npm start
```

此时系统会:
1. 仅使用本地SQLite
2. 无需网络连接
3. 无需配置云密钥
4. 数据访问速度最快

### 场景3: 多人协作(云模式)

```bash
# 设置云模式
export STORAGE_MODE=cloud

# 启动服务
npm start
```

此时系统会:
1. 所有数据存储在云端
2. 多人实时共享数据
3. 通过云开发控制台管理
4. 需要稳定的网络连接

## 切换注意事项

### 从 local 切换到 cloud/hybrid

切换时需要:
1. 配置腾讯云API密钥
2. 手动执行一次全量同步:
   ```bash
   curl -X POST http://localhost:3000/api/cloud/sync/all
   ```

### 从 cloud 切换到 local

切换前建议:
1. 备份云端数据到本地:
   ```bash
   curl -X POST http://localhost:3000/api/cloud/restore/customers
   curl -X POST http://localhost:3000/api/cloud/restore/products
   # ... 恢复其他表
   ```

### 从 hybrid 切换到 local

切换前建议:
1. 暂停自动同步
2. 确认本地数据完整
3. 备份本地数据库文件

## 故障排查

### 问题1: 切换到云模式失败

**错误**: 云存储初始化失败

**解决方法:**
1. 检查网络连接
2. 确认API密钥是否正确
3. 检查环境ID是否有效
4. 查看腾讯云配额是否充足

### 问题2: 混合模式同步失败

**错误**: 自动同步失败

**解决方法:**
1. 检查网络连接
2. 手动触发同步:
   ```bash
   curl -X POST http://localhost:3000/api/cloud/sync/incremental
   ```
3. 查看错误日志
4. 考虑切换到本地模式

### 问题3: 云模式下数据访问慢

**原因**: 网络延迟

**解决方法:**
1. 切换到混合模式,享受本地访问速度
2. 优化网络环境
3. 使用更近的云服务区域

## 推荐配置

### 个人使用
推荐使用 **混合模式 (hybrid)**
- 本地访问快速
- 云端自动备份
- 离线可用
- 数据安全有保障

### 团队协作
推荐使用 **云模式 (cloud)**
- 数据实时共享
- 集中管理
- 云端控制台方便

### 开发测试
推荐使用 **本地模式 (local)**
- 快速开发
- 无需配置
- 隔离测试环境

## 总结

存储模式切换功能让系统更加灵活:

1. **local模式** - 快速、简单、无网络依赖
2. **cloud模式** - 共享、集中、远程访问
3. **hybrid模式** - 兼顾速度和备份

根据实际需求选择合适的模式,可以随时切换!
